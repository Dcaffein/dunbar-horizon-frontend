# Task 26: Flag UI/UX 개선

## 배경

스크린샷 캡처 후 UI 검토를 통해 발견한 Flag 관련 개선사항을 일괄 반영한다.

---

## 변경 상세

### `/flags` 목록 페이지

| 항목 | 현재 | 변경 |
|---|---|---|
| 탭 순서·이름 | 주최 중 / 참여 중 / 친구 Flag | **둘러보기 / 호스팅 / 참여 중** |
| 종료 Flag | 목록에 표시 | 기본 숨김 |
| 모집 상태 필터 | 없음 | "종료 포함" 토글 — 기본 OFF (모집 중만 표시) |
| Encore 버튼 | "Encore" | "앙코르" |
| 삭제 버튼 스타일 | 일반 버튼과 동일 | 빨간색 계열 destructive 스타일 |
| 카드 description | 미표시 | 표시 (1~2줄 truncate) |
| 날짜 표기 | "6. 11. 오후 04:17" | "2026. 6. 11. 16:17 ~ 18:17" (시작~종료 범위) |

### `/flags/new` 생성 폼

| 항목 | 현재 | 변경 |
|---|---|---|
| 취소 버튼 | 없음 (← 뒤로가기만 가능) | [취소] 버튼 추가 → `/flags`로 이동 |
| 시작/종료 일시 기본값 | 현재 시각 자동 채움 | 빈 상태 (사용자가 직접 선택) |
| 모집 마감 필드명 | "모집 마감일" | "모집 마감" |
| 모집 마감 input 타입 | `date` (날짜만) | `datetime-local` (날짜+시간) |

### `/flags/{id}` 상세 페이지

| 항목 | 현재 | 변경 |
|---|---|---|
| 종료 Flag의 [수정] 버튼 | 노출 | 숨김 (종료 status일 때) |
| "기억 남기기" 섹션명 | "기억 남기기" | "메모리얼 남기기" |
| Memorial placeholder | 잘림 ("F"만 보임) | 전체 텍스트 표시 |
| 댓글 비공개 UI | 🔒 아이콘 | 잠금 아이콘 토글 — 🔓 공개 / 🔒 비공개 · 발신자에게만 표시 (Task 29와 동일 방식) |
| 정원 표시 | "N명 참여 중" | "참여자 N명 / 최대 N명" |
| 레이아웃 | 전체 너비 | 중앙 좁게 (목록 페이지와 통일) |

| `src/app/flags/new/page.tsx` 또는 `FlagForm.tsx` | 취소 버튼·기본값 제거·모집 마감 datetime-local |
| `src/components/Flag/FlagDetail.tsx` | 수정 버튼 조건부 노출·메모리얼 섹션명·placeholder·댓글 비공개 UI·정원 표시·레이아웃 |

## 검증

### Phase 1
- `npx tsc --noEmit` 에러 없음
- `npm run lint` 에러 없음

### Phase 2
- `/flags` 주최 중 탭이 "모집 중"으로 표시
- 종료된 Flag가 목록에 노출되지 않음
- 모집 중 필터 토글 동작
- 버튼명 "앙코르" 표시
- 삭제 버튼 빨간색 계열
- 카드에 description 1~2줄 표시
- 날짜 시작~종료 범위 포맷 확인
- `/flags/new` [취소] 버튼 → `/flags` 이동
- 시작/종료 일시 기본값 없음
- 모집 마감 datetime-local 입력 동작
- `/flags/{id}` 종료 Flag → [수정] 버튼 없음
- 메모리얼 섹션명 "메모리얼 남기기"
- Memorial placeholder 전체 표시
- 댓글 비공개: 🔓/🔒 아이콘 토글 + 텍스트 레이블 동작
- 정원 "참여자 N명 / 최대 N명" 표시
- 상세 페이지 중앙 좁게 레이아웃
