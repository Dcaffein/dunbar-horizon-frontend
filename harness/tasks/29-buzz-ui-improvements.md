# Task 29: Buzz UI/UX 개선

## 배경

스크린샷 캡처 후 UI 검토를 통해 발견한 Buzz 관련 개선사항을 일괄 반영한다.
비공개 댓글 잠금 아이콘 토글 방식은 Flag(`/flags/{id}`)와 동일하게 통일한다.
Flag의 비공개 댓글 UI 변경은 **Task 26**에서 처리한다.

---

## 변경 상세

### `/buzzes` 목록 페이지 — `BuzzList.tsx`

| 항목 | 현재 | 변경 |
|---|---|---|
| 레이아웃 | 전체 너비 | 중앙 좁게 (`max-w-2xl mx-auto`) |

### `/buzzes/new` 생성 폼 — `BuzzForm.tsx`

| 항목 | 현재 | 변경 |
|---|---|---|
| 탭 이름 | `Anchor` (영어) | `수신자 추천` |

| 설명 문구 | 없음 | `"선택한 친구를 기준으로 관련도 높은 친구들이 자동으로 수신자로 등록됩니다."` (text-xs text-gray-400) |
| 파일 업로드 | `<input type="file">` 브라우저 기본 UI | 커스텀 버튼 — `[📎 이미지 첨부]` 버튼 클릭 → hidden input trigger, 선택된 파일명/개수 표시 |

### `/buzzes/{id}` 상세 페이지 — `BuzzDetail.tsx`

| 항목 | 현재 | 변경 |
|---|---|---|
| 레이아웃 | 전체 너비 | 중앙 좁게 (`max-w-2xl mx-auto`) |
| 비공개 댓글 UI | `<input type="checkbox"> 공개` | 잠금 아이콘 토글 (아래 상세 참조) |

#### 비공개 댓글 토글 상세

댓글 입력창 하단 좌측에 배치. 클릭 시 상태 전환.

```
공개 상태 (기본):
  🔓 공개

비공개 상태 (클릭 후):
  🔒 비공개 · 작성자에게만 표시
```

- 아이콘 + 텍스트 레이블 항상 함께 표시
- 기본값: 공개 (`isPrivate = false`)
- 전송 시 현재 상태를 API에 전달 (기존 `isPrivate` 필드 재사용)

## 검증

### Phase 1
- `npx tsc --noEmit` 에러 없음
- `npm run lint` 에러 없음

### Phase 2
- `/buzzes` 목록이 중앙 좁게 표시됨
- `/buzzes/new` 첫 번째 탭이 "수신자 추천"으로 표시
- "기준 친구" 레이블 + 설명 문구가 드롭다운 위에 표시
- 이미지 첨부 버튼 클릭 → 파일 선택 다이얼로그 열림
- 파일 선택 후 파일명/개수 텍스트로 표시
- `/buzzes/{id}` 상세가 중앙 좁게 표시됨
- 댓글 입력창 아래 "🔓 공개" 기본 표시
- 클릭 시 "🔒 비공개 · 작성자에게만 표시"로 전환
- 비공개 상태로 전송 → 해당 댓글이 작성자 외 수신자에게 안 보임

### Phase 3
- 파일 선택 후 취소(다시 클릭하여 파일 제거) → 상태 초기화
- 비공개 댓글 전송 후 페이지 새로고침 → 비공개 상태 유지 표시
