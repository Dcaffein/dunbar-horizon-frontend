# Task 22: 경로 탐색

## 배경

친밀도(대칭) 또는 관심도(비대칭)를 엣지 가중치로 삼아
나와 특정 유저 사이의 "가장 신뢰도 높은 경로"와 중개인을 찾는 기능.

단순 최단 경로가 아닌 **가중치 기반 최적 경로** — 공통 친구를 통한 연결 강도를 시각화한다.

## 예시

```
나 ──(친밀도 0.9)── 박민준
나 ──(0.7)── 김철수 ──(0.95)── 박민준
→ 김철수를 통한 경로가 가중치 합산으로 더 강한 연결일 수 있음
```

## 선행 조건

**백엔드 경로 탐색 API 추가 필요.**

프론트에서 전체 친구 그래프를 순회하는 방식은 친구 수가 늘어나면 비현실적이므로
백엔드에서 계산 후 경로 결과를 반환하는 API가 필요하다.

예상 API 형태:
```
GET /api/v1/networks/path?targetId={id}&metric=intimacy|interest
→ [{ userId, nickname, ... }]  // 나 → 중개인들 → 목적지 순서
```

## UI 위치

- 친구 프로필 페이지(`/friends/{friendId}`) 내 "우리를 연결하는 경로" 섹션
- 또는 2-hop 추천 결과(Task 08)에서 "이 사람에게 닿는 경로" 표시

## 스코프 외

- 경로 탐색 알고리즘 (백엔드 담당)
- 여러 경로 비교 표시
